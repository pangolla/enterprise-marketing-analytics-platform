## Step 2 — Data model and relationships

Most CSVs are **pre-aggregated** at different grains. You do not need a complex star schema to build this portfolio dashboard.

### Recommended approach (simple — start here)

- Use each table **directly** in visuals on its page.
- **Do not** relate `Executive KPIs` to detail tables (it is a single-row snapshot).
- Page-level slicers filter only the table they are built from.

### Optional relationships (for cross-filtering)

Create these only if you want slicers to flow across visuals:

| From table | From column | To table | To column | Cardinality | Cross-filter |
|------------|-------------|----------|-----------|-------------|--------------|
| Revenue by Month | `year`, `month` | Delivery Performance | `year`, `month` | Many:1 | Single |
| Revenue by Category | `category_en` | Category Pareto | `category_en` | 1:1 | Both |
| Revenue by Category | `category_en` | Category Monthly Trend | `category_en` | 1:Many | Single |
| Segment Summary | `segment_name` | Customer Segments | `segment_name` | 1:Many | Single |
| Revenue by State | `customer_state` | Customer Segments | `customer_state` | 1:Many | Single |

**Create a date helper column** (optional) in both `Revenue by Month` and `Delivery Performance`:

```powerquery
// Power Query custom column
DateKey = [year] * 100 + [month]
```

Relate on `DateKey` instead of two columns.

### Tables that stay standalone

| Table | Why |
|-------|-----|
| `Executive KPIs` | Single-row snapshot — use for cards only |
| `Repeat Purchase KPI` | Single-row |
| `Delivery Delay KPI` | Single-row |
| `Delay vs Review` | 2 rows only |
| `Review Summary` | Aggregate review distribution |

## Step 3 — Dashboard theme and formatting

Apply before building visuals so all pages match.

### Canvas

1. **View → Page view → Fit to page**.
2. **Format page → Canvas settings**:
   - Type: **16:9**
   - Width: **1280** px (or 1920 for higher-res screenshots)
   - Background: **#FAFAFA** (off-white)

### Theme colors

| Role | Hex | Use |
|------|-----|-----|
| Background | `#FAFAFA` | Page canvas |
| Text | `#1A1A2E` | Titles, labels |
| Primary accent | `#0D9488` | Teal — lines, primary bars |
| Secondary accent | `#059669` | Green — highlights, KPI accents |
| Alert / delay | `#DC2626` | Delay rate, negative signals |
| Neutral grid | `#E5E7EB` | Chart gridlines |

### Fonts

- **Segoe UI** or **Calibri**
- Title: 14–16 pt, bold, `#1A1A2E`
- KPI card value: 28–32 pt
- Axis labels: 10–11 pt

### Number formats (apply per visual)

| Field type | Format | Example |
|------------|--------|---------|
| Revenue | Custom: `R$ #,##0` | R$ 13,221,498 |
| AOV | `R$ #,##0.00` | R$ 137.04 |
| Percentages | `0.0"%"` | 3.0% |
| Review score | `0.00` | 4.09 |
| Counts | `#,##0` | 96,478 |

### Visual defaults

- Border: Off or subtle `#E5E7EB`
- Padding between visuals: 8–12 px
- Align KPI cards in a single row at top of Page 1

**Sort axis trick:**
1. Add `year` and `month` to X-axis.
2. Or create a calculated column: `period = year * 100 + month` and sort by it.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| KPI card shows blank | Check table is `Executive KPIs`; field aggregation = Sum or Don't summarize |
| Revenue shows billions | Wrong aggregation (Average instead of Sum) on multi-row table |
| Months sort alphabetically | Sort by `month` number, not `month_name` |
| Map does not work for Brazil | Use bar chart instead of filled map |
| Slicer does not filter KPI cards | Expected — snapshot table is not related to detail tables |
| CSV not found | Re-run pipeline or check `data/processed/dashboard_exports/` |
